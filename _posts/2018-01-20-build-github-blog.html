网上借鉴了N个分享之后整理出来的，只有基础页面，当前的github版本可以用，分享一下哈
*username.github.iomaster分支与任意项目下的gh-pages分之区别：username.github.io，当用户访问这个域名时， GitHub会去解析username用户下，username.github.io项目的master分支；你有一个项目名为blog， GitHub为这个项目提供的域名username.github.io/blog， 当你访问这个域名时，GitHub会去解析username用户下，blog项目的gh-pages 分支。 所以，要搭建自己的博客，你可以选择建立名为 username.github.io的项目， 在master分支下存放网站源代码，也可以选择建立名为 blog 的项目，在 gh-pages分支下存放网站源代码 当前的是新建blog项目，github升级之后将github pages设置为master branch 1、申请一个github账号
2、Create a new repository，名字随便
3、点击项目，settings
4、clone代码，git clone your_repository_url
5、cd 文件夹，新建文件Gemfile，写上如下代码
source 'https://rubygems.org'
gem "github-pages", group: :jekyll_plugins
执行bundle install，不成功就bundle update
删除目录下的所有文件，git相关可以保留
6、jekyll new ./
7、jekyll serve查看效果，浏览器中输入http://127.0.0.1:4000/
8、为了兼容Github默认的Jekyll，需要把
gem "jekyll", "3.4.3"替换成gem "github-pages", group: :jekyll_plugins
注意：测试还用原来的gem，上传改，上传的时候可以忽略
9、_config.yml，修改
url和baseurl，baseurl为你建的项目名字，baseurl: "/blog"
url为你的账号默认的访问地址，url: "https://mingtaowang.github.io"
10、推送更改
git add .
git commit -m 'jekyll'
git push或者git push origin master:master
11、访问https://yourusername.github.io/blog（为你项目的名字）/
eg：https://mingtaowang.github.io/blog/
12、再然后，就可以填东西了，本地测试看效果，jekyll serve，然后推上去。
大家有更好的方法可以告诉我哦，可以看一下
